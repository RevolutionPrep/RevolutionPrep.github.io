Clients must authorize using basic http authentication in order to access any of the private API's listed on this page.

Connect with the Revolution Prep development team in order to access the current credentials.

Index

URL: https://console.revolutionprep.com/api/v1/private/instructors.json
Method: GET

Optional Parameters
start_date
end_date

Limit results to only those who have been updated within the start_date and end_date. This filter can be used in conjunction with the ids parameter.

Note: start_date and end_date must both be passed to filter by updated at date range.

?start_date=2014-01-01&end_date=2014-12-31

The above example will only return records that were updated between January 1, 2014 and December 31, 2014.
ids:

Limit by instructor ids.

Parameter value should be a comma separated list of ids

Example: ids=1,2,3
page:

an integer value which specifies which page of results to fetch, starting at 1. Fetching successively higher page numbers will return additional results, until there are no results to return (in which case an empty result set will be returned). Defaults to 1 page and 1000 records per page.

Attributes:

  • current_page: Current page # of results
  • total_entries: Total number of results for the given query
  • total_pages: Total number of pages of results
  • instructors: List of instructors
    • id: Id of the instructor record
    • created_at: Creation date of the record
    • updated_at: Last updated performed at
    • first_name: First name of the instructor
    • last_name: Last name that the instructor
    • email: Email of the instrucor
    • phone: Phone of the instructor
    • address: Hash of the instructor address
      • street_1
      • street_2
      • city
      • state
      • zip
    • profile: Hash of the instructor profile attributes
      • university: University name that the instructor attended
      • major: Major of instructor
      • degree: Degree of instructor
      • last_score_percentile: Percentage of instructor score on the SAT or ACT
      • star_rating: The average star rating (out of 5 stars) from student reviews
      • biography: Biography of the instructor
      • avatar_url: Url to retrieve the headshot of the instructor

JSON example

{% highlight javascript %} https://console.revolutionprep.com/api/v1/private/instructors.json { current_page: 1, total_entries: 117, total_pages: 1, instructors: [ { id: 3038, created_at: "2013-10-17T08:06:01-07:00", updated_at: "2014-04-09T11:28:32-07:00", first_name: "Stanis", last_name: "Tsibulsky", email: "stanis.tsibulsky@gmail.com", phone: "333-222-4444", address: { street_1: "1234 Main Street", street_2: null, city: "Chicago", state: "IL", zip: "60622" }, profile: { university: "University of Michigan", major: "History of Art, Contemporary Asian Studies", degree: "BA, MSC", last_score_percentile: null, star_rating: 4.7412, biography: "Having taken many AP classes in high school, Stanis...", avatar_url: "https://s3.amazonaws.com/profile_attachments/9732/stanis_cropped.jpg" } } ] } {% endhighlight %}

Proposals

Index

URL: https://console.revolutionprep.com/api/v1/private/proposals.json
Method: GET

Optional Parameters
start_date
end_date

Limit results to only those who have been updated within the start_date and end_date. This filter can be used in conjunction with the ids parameter.

Note: start_date and end_date must both be passed to filter by updated at date range.

?start_date=2014-01-01&end_date=2014-12-31

The above example will only return records that were updated between January 1, 2014 and December 31, 2014.
ids:

Limit by proposal ids.

Parameter value should be a comma separated list of ids

Example: ids=1,2,3
page:

an integer value which specifies which page of results to fetch, starting at 1. Fetching successively higher page numbers will return additional results, until there are no results to return (in which case an empty result set will be returned). Defaults to 1 page and 1000 records per page.

Attributes:

  • current_page: Current page # of results
  • total_entries: Total number of results for the given query
  • total_pages: Total number of pages of results
  • proposals: List of proposals
    • id: Id of the proposal record
    • created_at: Creation date of the proposal
    • updated_at: Last updated performed at
    • type: Type of the proposal: proposal or re-up. Type proposal is for a new program of study. This type will only have a program_id in won state. A re-up is a proposal to add hours to an existing program of study. As such, a program_id will be set in all states of a re-up proposal.
    • state: State of proposal. Will be one of the following: open, lost, or won
    • hours: Total proposed hours of instruction
    • full_price: Full price of the proposal without any discounts (in USD and will not include currency formatting)
    • discount: Amount to be discounted off full price (in USD and will not include currency formatting)
    • net_price: Full price minus any discount (in USD and will not include currency formatting)
    • delivery_method: in-person or online
    • brand: Brand of proposed program
    • segment: Segment of the proposed program
    • subject: Subject of the proposed program
    • student_id: ID of the student associated with the proposal
    • instructor_id: ID of the instructor associated with the proposal
    • program_id: ID of the program a won proposal was delivered into. Proposals of type 'proposal' will only have a program_id if the proposal is in state 'won'. If the proposal type is 're-up', a program_id will be set in all states.

JSON example

{% highlight javascript %} https://console.revolutionprep.com/api/v1/private/proposals.json { current_page: 1, total_entries: 1, total_pages: 1, proposals: [ { id: 106078, created_at: "2014-07-02T10:38:53-07:00", updated_at: "2014-07-02T11:18:53-07:00", state: "won", hours: 10, price: "1000.0", delivery_method: "online", brand: "Private Tutoring", segment: "Standard", subject: "SAT", student_id: 123456, instructor_id: 1234, program_id: 210292 } ] } {% endhighlight %}

Create

URL: https://console.revolutionprep.com/api/v1/private/notes.json
Method: POST

Required Parameters
console_id
user_type

The console record id the note is intended for. Must be accompanied by the user_type (student, parent, employee).
message:

The content of the note.
employee_id:

The console employee ID of the writer of the note.
Response
Successful

If the note is successfully created, the response status will be 200 and will render the newly created notes id as json:

:id => note.id
Failure

If the note cannot be created, the response status will be 422 and will render json:

:error => "Reason note could not be created"

Update

URL: https://console.revolutionprep.com/api/v1/private/note/:note_id.json
Method: PUT

Required Parameters
:id

The console note_id of the note to be updated.
message:

The content of the note.
employee_id:

The console employee ID of the writer/editor of the note. This will overwrite the employee_id.
Response
Successful

If the note is successfully created, the response status will be 200. There will be no response json rendered.
Failure

If the note cannot be created, the response status will be 422 and will render json:

:error => "Reason note could not be created"
